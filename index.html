HTML

  • Hypertext Markup Language (HTML), to give the content of a webpage structure
  • The head element contains information about the webpage.
  • The head contains Metadata, Metadata is data about our webpage that the browser uses to effectively serve our webpage to users
  • The body element represents the visible content shown to the user.
  • All the elements that we want the user to see and interact with must be placed inside the body.

CSS

  • Cascading Style Sheets (CSS), to give the contents style. It lets us change the style, position and overall look of the website
  • A margin indicates how much space we want around the outside of an element.
  • The margin is not visible on the page; it merely adds a buffer (When it is 0, there is no space around the element)
  • A padding indicates how much space we want around the content inside an element.
  • The order is content,padding,border and margin

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git pull origin main, pulls the most up to date version from the repo to the main branch of your local repo
  • git add -A adds modifications in the current working branch to the staging area
  • git commit -m "" details what the staged files to be committed are
  • git push origin (your branch name) pushes the changes we have made locally to our remote branch

JavaScript

  • JavaScript is a scripting or programming language that allows you to implement complex features on web pages
  • This is accomplished by creating logic via the use of e.g variables, arrays, for loops and functions
  • Javascript is placed in a script html tag in the body, it is best to link an external js file
  • a function is simply a set of instructions that tells the computer how to perform a certain task. They are not automatically execute but instead need to be run
  • when using the if statement, code will only execute if the statement meets a condition, or is true.
